Meta Description Added to All Pages
CRITICALOn-Page SEOCTR +20–35%
What Changed

A meta description is the short paragraph under your website title in Google search results. gsurf.in has none — so Google generates one from random page text (often the modal popup HTML), which looks unprofessional and irrelevant to users searching for broadband.

Why It Matters
SEOMeta description is a direct CTR ranking factor. Higher CTR sends a stronger behavioral signal to Google, which compounds into ranking improvements over time.
AEOAI answer engines (ChatGPT, Perplexity, Gemini) read meta description as the primary "what is this page about" signal when deciding whether to cite this page in an answer.
GEOCity name in meta description reinforces geographic relevance for Bangalore/Whitefield local queries — a direct geo signal for local intent matching.
🔴 CRITICAL · CTR impact: ~20–35%

Structured Data (JSON-LD Schema) — Organization + LocalBusiness + FAQ
CRITICALStructured DataAI citation eligibility
What Changed

Schema markup is code that tells search engines and AI tools exactly what type of business this is, where it operates, what it offers, and who to contact. Currently zero schema exists on gsurf.in — the site is completely invisible to AI tools and ineligible for rich results.

Why It Matters
SEOSchema enables rich results — FAQ boxes, LocalBusiness cards, breadcrumbs. Rich results get 20–30% more clicks than plain blue links.
AEOThe #1 AEO fix. AI engines like Perplexity and ChatGPT use structured data to extract factual information for their answers. Without schema, GSurf cannot be cited by any AI tool.
GEOLocalBusiness + ServiceArea + areaServed schema is how Google AI Overviews and Gemini determine which ISPs serve which neighbourhoods.
🔴 CRITICAL · AI invisibility without this

Heading Hierarchy Fixed — One Keyword-Rich H1 Per Page
CRITICALOn-Page SEOPrimary keyword signal
What Changed

The page has three H1 tags and none contain searchable keywords. "Ultra Fast Internet! Better Service!!" has zero keyword value. "We Provide Unlimited Network For You" is marketing fluff. "Enterprise Broadband Plans" inside a div is structurally wrong. One keyword-rich H1 per page is a fundamental SEO rule — Google uses it as the strongest on-page content signal after the title tag.

Why It Matters
SEOH1 = primary topic declaration to Google. Must contain the main target keyword. Multiple H1s signal disorganized, low-quality content.
AEOAI tools use heading structure to understand page topic. A clear keyword-rich H1 helps AI extract the right entity description for citation.
GEOCity/area pages (HSR Layout, BTM Layout) must include location in H1: "Fiber Broadband in HSR Layout, Bangalore" — this is a direct geo-relevance signal.
🔴 CRITICAL · Core ranking signal

GEOog:locale="en_IN" signals to AI tools this is an India-specific resource. Combined with address schema, strengthens local geo-relevance significantly.
🟠 HIGH · Social + AI entity signals

Multi-Page Architecture — 21 Separate Rankable URLs
CRITICALTechnical SEOStructural foundation
What Changed

The entire website was one HTML file with anchor links (#home, #plans, #contact). Google ranks pages, not sections. The revamp creates 21 separate pages each with a unique URL, each targetable for different search queries. This is the most important structural change in the entire project.

Why It Matters
SEO21 pages = 21 independent ranking opportunities vs 1. Each page can target a specific query: "fiber broadband plans Bangalore," "fiber internet HSR Layout," "business broadband Whitefield."
AEOAI tools cite specific URLs, not websites. A page dedicated to "GSurf Bangalore fiber plans" is cited by AI answering "best fiber broadband in Bangalore." A homepage section never is.
GEOArea pages (HSR Layout, BTM Layout) rank for neighbourhood-level queries with high conversion intent that a homepage can never compete for.
🔴 CRITICAL · Root structural fix

07
robots.txt + XML Sitemap Created
HIGHTechnical SEOIndex coverage
What Changed

Both files returned 404 on the live site. robots.txt tells bots which pages to crawl. Without it, Google wastes crawl budget on modal HTML, form elements, and the portal.gaxiom.in redirect. sitemap.xml lists all 21 revamped pages — without it, deeper pages like /pages/hsr-layout/ may take weeks to be discovered.

Why It Matters
SEOSitemap ensures all 21 pages get crawled quickly. Submit to Google Search Console on launch day to guarantee rapid indexing.
AEOBing's crawler (used by Perplexity and Copilot) respects robots.txt and uses sitemap.xml to discover pages. Pages not in sitemap are rarely indexed by AI tools.
GEOLocation pages (/pages/bangalore/, /pages/hsr-layout/) must appear in sitemap to rank for local neighbourhood queries.
🟠 HIGH · Required infrastructure

CRITICALLocal SEOLocal Pack eligibility
What Changed

NAP = Name, Address, Phone. Currently inconsistent across the page (4 brand names) and completely unstructured (plain text, no schema). Contact emails are obfuscated by Cloudflare and invisible to crawlers. The fix applies consistent brand identity and wraps NAP in machine-readable schema + HTML microdata.

SEONAP consistency across website + GBP + directories is the #1 local ranking factor for Local Pack positions.
AEOAI tools build entity knowledge from consistent NAP. Inconsistency = AI cannot build a reliable entity profile = no citation in AI-generated answers.
GEOThis is the foundation of GEO. AI-powered local answers pull business data from structured, consistent NAP signals to determine who serves which area.

FAQPage Schema + FAQ Content Section — AEO Implementation
HIGHAEO / GEOAI citation + rich results
What Changed

Zero FAQ content existed on any page. AEO requires content structured as natural language Q&A so AI tools can extract and present it as answers. When someone asks "How much does GSurf cost?" or "Is GSurf in HSR Layout?" — there was nothing on the site structured for AI to cite. Both visible FAQ HTML and FAQPage JSON-LD schema are required.

Why It Matters
SEOFAQPage schema generates expandable FAQ rich results in Google SERPs — takes up 3× more SERP space, gets 20–30% more clicks than plain results.
AEOPure AEO. Perplexity, ChatGPT, Gemini extract from FAQ-formatted content. A Q&A answering "How much does GSurf cost?" = GSurf gets cited in AI answers about pricing.
GEOLocation-specific FAQs ("Is GSurf in HSR Layout?") train AI on geographic coverage, making GSurf appear in neighbourhood-specific AI answers.

BreadcrumbList Schema — City > Area Hierarchy in SERPs
HIGHStructured DataSERP real estate
What Changed

With the multi-page hierarchy (Bangalore → HSR Layout, Bangalore → BTM Layout), BreadcrumbList schema enables the breadcrumb trail to appear directly in Google search results below the page URL — showing "GSurf › Bangalore › HSR Layout" before users even click. Required on all city and area drilldown pages.

Why It Matters
SEOBreadcrumb rich results increase SERP click-through rates by showing site structure. Users trust hierarchical results more than standalone URLs.
AEOBreadcrumb schema helps AI tools understand the content hierarchy — that HSR Layout is a sub-location of Bangalore — strengthening geographic entity relationships.
GEOThe Bangalore › HSR Layout relationship in schema teaches AI tools the geographic hierarchy used for local search and recommendation queries.
🟠 HIGH · SERP hierarchy signals
